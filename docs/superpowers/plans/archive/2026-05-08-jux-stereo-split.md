# `.jux(f)` Implementation Plan

**Status:** Shipped; archived after implementation landed on `main`.

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `.jux(f)` combinator to moondsp's pattern language so that `s("bd sd hh sd").jux(rev)` plays the original pattern hard-left and `f(original)` hard-right, both stacked.

**Architecture:** Pure pattern-layer combinator. Build `left = merge_control(self, s_pan(-1.0))`, `right = merge_control(f(self), s_pan(1.0))`, return `stack([left, right])`. Stereo routing is already end-to-end (per-event `pan` → `SetPan` voice action → `VoiceSlot::update_pan` → `StereoOut`), so no DSP/scheduler/worklet changes are needed. The mini parser shares its `Callback` grammar between `.every` and `.jux` after a rename of the PR #17 helpers.

**Tech Stack:** MoonBit (`pattern/`, `mini/`, `scheduler/`, `browser/`), Vite/Playwright (`web/live/`).

**Spec:** `docs/superpowers/specs/archive/2026-05-08-jux-stereo-split-design.md`

```moonbit
///|
/// Parse the callback argument for `.every(n, f)` or `.jux(f)`.
/// Supported forms: `fast(N)` | `slow(N)` | `rev` | `rev()`.
///
/// `calling_method` is the user-visible method name (`"every"` or
/// `"jux"`) used in error messages — keeps the parse error specific
/// to whichever method the user actually wrote.
fn Parser::parse_callback(
  self : Parser,
  method_start : Int,
  name : String,
  calling_method~ : String,
) -> Callback raise ParseError {
  match name {
    "fast" => {
      self.expect_char('(')
      self.skip_whitespace()
      let k_dbl = self.read_number()
      self.skip_whitespace()
      self.expect_char(')')
      let k = require_integer(
        k_dbl,
        method_start,
        "\{calling_method}(fast(k))",
      )
      guard k > 0 else {
        raise ParseError(
          "position \{method_start}: \{calling_method}(fast(k)) requires a positive integer, got \{k}",
        )
      }
      make_fast(@pattern.Rational::from_int(k))
    }
    "slow" => {
      self.expect_char('(')
      self.skip_whitespace()
      let k_dbl = self.read_number()
      self.skip_whitespace()
      self.expect_char(')')
      let k = require_integer(
        k_dbl,
        method_start,
        "\{calling_method}(slow(k))",
      )
      guard k > 0 else {
        raise ParseError(
          "position \{method_start}: \{calling_method}(slow(k)) requires a positive integer, got \{k}",
        )
      }
      make_slow(@pattern.Rational::from_int(k))
    }
    "rev" => {
      // Accept both `rev` and `rev()` as the callback form.
      if !self.at_end() && self.input[self.pos] == '(' {
        self.expect_char('(')
        self.expect_char(')')
      }
      make_rev()
    }
    _ =>
      raise ParseError(
        "position \{method_start}: \{calling_method}() callback must be fast(N), slow(N), or rev; got '\{name}'",
      )
  }
}
```

test "jux(rev): channel identity (pan=-1 matches self, pan=+1 matches f(self))" {
  // Use a multi-key pattern so the comparison loop catches any bug that
  // drops a control key while routing through merge_control. Without
  // a non-pan secondary key, the test only proves part+sound parity.
  let pat = merge_control(
    sequence([sound(36.0), sound(38.0)]),
    s_cutoff(800.0),
  )
  let result = pat.jux(fn(p) { p.rev() })
  let arc = span(0, 1, 1, 1)
  let combined = result.query(arc)
  let self_events = pat.query(arc)
  let f_events = pat.rev().query(arc)
  // Filter combined by pan key
  let left : Array[Event[ControlMap]] = []
  let right : Array[Event[ControlMap]] = []
  for ev in combined {
    match ev.value.0.get("pan") {
      Some(-1.0) => left.push(ev)
      Some(1.0) => right.push(ev)
      _ => ()
    }
  }
  // Compare key-by-key for sound and cutoff (every non-pan key from
  // the input). If jux drops or mangles any of them, this fails.
  assert_eq(left.length(), self_events.length())
  for i in 0..<left.length() {
    assert_eq(left[i].part, self_events[i].part)
    assert_eq(left[i].value.0.get("sound"), self_events[i].value.0.get("sound"))
    assert_eq(left[i].value.0.get("cutoff"), self_events[i].value.0.get("cutoff"))
  }
  assert_eq(right.length(), f_events.length())
  for i in 0..<right.length() {
    assert_eq(right[i].part, f_events[i].part)
    assert_eq(right[i].value.0.get("sound"), f_events[i].value.0.get("sound"))
    assert_eq(right[i].value.0.get("cutoff"), f_events[i].value.0.get("cutoff"))

///|
/// Strudel-style stereo split. Plays `self` panned hard left and
/// `f(self)` panned hard right, both stacked. Pre-existing `pan`
/// values on the input are overridden by the ±1 outer pan (Strudel-
/// correct: `jux` sets pan unconditionally).
///
/// Doubles per-pool concurrency only when `f` causes same-sound
/// coincidence (e.g. `fast(2)` packs the right channel denser, or
/// `rev` over a single-sound layer like `hh*16` puts left's hh at
/// slot t and right's hh at slot 1-t). For sparse multi-sound
/// patterns like `s("bd sd hh sd").jux(rev)` the per-pool pressure
/// stays at 1.
pub fn Pat::jux(
  self : Pat[ControlMap],
  f : (Pat[ControlMap]) -> Pat[ControlMap],
) -> Pat[ControlMap] {
  let left = merge_control(self, s_pan(-1.0))
  let right = merge_control(f(self), s_pan(1.0))
  stack([left, right])
}
```

## Task 4: Scheduler-level stereo regression tests

**Note: this is a characterization task, not TDD.** Stereo routing is already implemented end-to-end (per-event `pan` → `SetPan` → `VoiceSlot::update_pan` → `StereoOut`); these tests *lock in* an existing contract that the previous pan test only proved by "voice was created". Codex flagged this in spec review — `.jux` depends on this routing invariant, so it's worth pinning down before we ship the combinator.

Run multiple `process_block` calls before asserting energy: a single 128-frame block at the start of an envelope can be near-zero. The existing `process_block: produces non-zero output` test in `scheduler_test.mbt` runs **4 blocks** before checking — we mirror that here. The strict assertion is on the *silent* side (right < ε for hard-left), which holds at any block count because pan_right_gain is exactly 0.

///|
/// Sum of absolute sample values across an `AudioBuffer`. Used to
/// detect whether a buffer carries any signal vs is silent.
fn buffer_abs_sum(buf : @moondsp.AudioBuffer) -> Double {
  let arr = buf.as_fixed_array()
  let mut total = 0.0
  for i in 0..<arr.length() {
    let v = arr[i]
    total = total + (if v < 0.0 { -v } else { v })
  }
  total
}

///|
test "process_block: pan=-1 routes signal to left buffer only" {
  let (pool, sched, left, right) = make_test_sched()
  let pat = @pattern.merge_control(@pattern.note(60.0), @pattern.s_pan(-1.0))
  // Run 4 blocks so the ADSR attack accumulates audible energy; matches
  // the existing `process_block: produces non-zero output` test shape.
  for _ in 0..<4 {
    sched.process_block(pat, pool, left, right)
  }
  let left_energy = buffer_abs_sum(left)
  let right_energy = buffer_abs_sum(right)
  // Strict assertion is on the silent side: pan_right_gain is exactly 0
  // for pan=-1, so right_energy is exactly 0 regardless of envelope phase.
  assert_eq(right_energy, 0.0)
  // Loose assertion on the active side: any non-zero proves routing fired.
  assert_true(left_energy > 0.0)
}

///|
test "process_block: pan=+1 routes signal to right buffer only" {
  let (pool, sched, left, right) = make_test_sched()
  let pat = @pattern.merge_control(@pattern.note(60.0), @pattern.s_pan(1.0))
  for _ in 0..<4 {
    sched.process_block(pat, pool, left, right)
  }
  let left_energy = buffer_abs_sum(left)
  let right_energy = buffer_abs_sum(right)
  assert_eq(left_energy, 0.0)
  assert_true(right_energy > 0.0)
}
```

packs density; jux(rev) over single-sound layers like hh*16? puts
left's hh at slot t and right's at slot 1-t).

Multi-sound .jux(rev) patterns don't need this headroom — included
only for the demo's hh*16? layer and similar dense single-sound
cases."
